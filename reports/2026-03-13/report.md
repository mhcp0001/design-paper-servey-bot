### 🥇 Rank 1

**Reframing Through New Minds: How External Experts Unlock Problem Reframing Through Reasoning Logics**

| 項目 | 内容 |
|------|------|
| 著者 | Paola Bellis et al. |
| ジャーナル | Journal of Product Innovation Management |
| 発表年月 | 2026-03 |
| 論文タイプ | empirical |
| DOI | https://doi.org/10.1111/jpim.70038 |
| OpenAlex ID | https://openalex.org/W7134858137 |
| OAステータス | hybrid |
| PDF/原文 | [DOI Landing Page](https://doi.org/10.1111/jpim.70038)（要機関アクセス） |

#### 一言要約
外部専門家が5種類の推論ロジック（逆演繹・逆帰納・説明的アブダクション・思索的アブダクション・道徳的アブダクション）を用いることで、組織が問題をリフレーミングし革新的な洞察を得るプロセスを明らかにしたエスノグラフィック研究。

#### 研究概要
- **背景**: イノベーションの初期段階における「問題リフレーミング」は創造的解決策の前提条件だが、そのメカニズムは未解明。外部専門家が組織内の固定観念を打破する役割を担うことが指摘されてきた。
- **手法**: エスノグラフィー。製品イノベーションプロジェクトへの外部専門家の関与を長期観察。
- **主要知見**: 外部専門家は5つの推論ロジック（逆演繹・逆帰納・説明的アブダクション・思索的アブダクション・道徳的アブダクション）を使いわけ、組織の問題解釈を根底から転換させる。各ロジックは異なる認知的経路で「問題の枠組み」を揺さぶる。

#### 実務的ポイント
- **ビジネスデザイン**: クライアントが「正しい問いを立てられていない」状況への介入手法として活用可能。ファシリテーター・外部コンサルタントが使うべき推論モードを意識的に設計できる。
- **戦略デザイン**: 長期ロードマップ策定時に「思索的アブダクション」（仮想シナリオからの逆算推論）を意図的に組み込むフレームワークとして応用可能。
- **ビジョンデザイン**: 未来洞察セッションで「道徳的アブダクション」（倫理的示唆からの逆算）を活用することで、技術的制約を超えた価値観主導のリフレーミングが実現できる。

#### 理論的貢献
- アブダクション推論を「種類別」に分類したことで、イノベーション初期段階の認知プロセス研究に新たな類型論を提供。
- 外部知識統合（absorptive capacity）研究と、問題フレーミング研究（Dorst, Cross）を架橋。

#### 限界・今後の研究課題
- エスノグラフィーのため一般化に制限あり。定量的検証が必要。
- 推論ロジックの使い分けを促す組織的要因（チーム構成、心理的安全等）は未検討。

#### 実践メモ
- **クライアントワーク**: 「なぜ問題をリフレームできないか」の診断ツールとして5ロジック分類を活用。
- **社内勉強会フレーミング**: 「外部視点はなぜ有効か？認知科学が解明するリフレーミングの構造」として紹介。

### 🥈 Rank 2

**Co-Design at the Boundary: Understanding the Dynamics of Open Innovation Between Companies and Communities**

| 項目 | 内容 |
|------|------|
| 著者 | Eric Reynolds Brubaker et al. |
| ジャーナル | Journal of Product Innovation Management |
| 発表年月 | 2026-03 |
| 論文タイプ | empirical |
| DOI | https://doi.org/10.1111/jpim.70020 |
| OpenAlex ID | https://openalex.org/W7134249052 |
| OAステータス | hybrid |
| PDF/原文 | [DOI Landing Page](https://doi.org/10.1111/jpim.70020)（要機関アクセス） |

#### 一言要約
自動車メーカーとオンラインコミュニティの5年間のコデザイン協働を追跡し、「親密型・手段型・限定型」の3形態が環境・組織要因により動態的に移行することを明らかにした、境界研究の最前線。

#### 研究概要
- **背景**: オープンイノベーションにおける企業-コミュニティ間のコデザインは、静的な状態ではなく動態的に変化するが、そのメカニズムは未解明。
- **手法**: 5年間のエスノグラフィック研究。自動車メーカー1社とオンラインカスタマーコミュニティの協働を追跡。
- **主要知見**: コデザインは「親密型（intimate）」「手段型（instrumental）」「限定型（limited）」の3形態に類型化でき、企業戦略・技術変化・コミュニティ文化の変化によって形態間を動態的に遷移する。遷移のトリガーとなる「駆動力」が特定された。

#### 実務的ポイント
- **ビジネスデザイン**: 顧客共創プロセスを「段階的ガバナンス設計」の視点で捉え直し、協働形態の意図的な設計・移行管理が可能になる。
- **戦略デザイン**: B2Bエコシステム設計において、パートナーとの関係性を「手段型→親密型」へと意図的に深化させる戦略立案に活用。
- **ビジョンデザイン**: 長期ビジョン策定において、社内外の利害関係者との共創関係の進化をロードマップに組み込む際のフレームとして有効。

#### 理論的貢献
- 「組織境界」研究に動態的視点を追加し、境界資源（boundary resources）の役割を実証。
- オープンイノベーション研究とコデザイン研究を統合する新たな分析枠組みを提供。

#### 限界・今後の研究課題
- 単一企業・単一コミュニティの事例のため、一般化には複数ケース研究が必要。
- デジタルプラットフォームへの応用可能性は未検討。

#### 実践メモ
- **クライアントワーク**: 顧客/パートナーとの協働プロジェクトの「関係性診断」に3形態分類を活用。
- **社内勉強会フレーミング**: 「5年間で変わる共創の姿：コデザイン3形態とその移行メカニズム」として紹介。

### 🥉 Rank 3

**Enhancing collaboration and knowledge sharing through intra-organizational platforms: a design science research study**

| 項目 | 内容 |
|------|------|
| 著者 | Silvia Gadola et al. |
| ジャーナル | Review of Managerial Science |
| 発表年月 | 2026-03 |
| 論文タイプ | empirical |
| DOI | https://doi.org/10.1007/s11846-026-00996-0 |
| OpenAlex ID | https://openalex.org/W7134073515 |
| OAステータス | hybrid |
| PDF/原文 | [PDF直リンク](https://link.springer.com/content/pdf/10.1007/s11846-026-00996-0.pdf) |

#### 一言要約
「プラットフォーム思考」を組織内部に適用し、既存企業が社内ユニット間の知識共有と協働を加速できることをデザインサイエンス研究アプローチで実証した、イントラ・プラットフォーム研究の先駆的論文。

#### 研究概要
- **背景**: プラットフォームは外部の顧客グループ間の効率的なマッチングによる価値創造で知られるが、同じ「プラットフォーム思考」を組織内部（社内ユニット間）に適用することでイノベーションが促進されるか、という問いは未検討だった。
- **手法**: デザインサイエンス研究（DSR）。ITベースの社内プラットフォームを設計・実装し、その効果を評価。研究サイクルを通じてプラットフォームアーキテクチャを反復改善。
- **対象**: 複数の社内ユニットを持つ企業（反復的なDSRサイクルにより設計原則を導出）。
- **主要知見**: 社内向けプラットフォームが「内部マッチングメカニズム」として機能し、従来の階層型・プロジェクト型組織では発生しにくかった「偶発的知識結合」を生み出すことを実証。DSRのアーティファクト設計として6つの設計原則を導出。

#### 実務的ポイント
- **ビジネスデザイン**: 大企業（日立のような多部門組織）における社内知識流通の設計に直結。「社内プラットフォーム」の設計原則として活用可能。
- **戦略デザイン**: 組織変革プロジェクト（デジタルワークプレイス再設計など）においてDSRアプローチを採用する際の方法論的支柱として機能。
- **ビジョンデザイン**: 10-20年の組織進化ロードマップにおいて「プラットフォーム型組織」への移行を位置づける際の理論的根拠として使用可能。

#### 理論的貢献
- 「プラットフォーム理論」を「外部向け」から「内部向け」へ拡張した初期段階の実証研究。
- デザインサイエンス研究とプラットフォーム理論を統合した新しい研究アジェンダを開拓。
- Eisenmann・Parker・Van Alstyne等のプラットフォーム研究に組織内部視点を追加。

#### 限界・今後の研究課題
- 単一のプラットフォーム設計事例のため、業種・組織規模への一般化には検討が必要。
- 長期的な組織学習効果（知識の蓄積・減衰）は研究スコープ外。

#### 実践メモ
- **クライアントワーク**: 「知識管理システムの導入」ではなく「社内プラットフォームの設計」として位置づけ直すことで、クライアントの組織変革プロジェクトの方向性を変えられる。
- **社内勉強会フレーミング**: 「なぜ社内SNSは使われないのか？プラットフォーム思考が解く組織内知識共有の謎」として紹介。
